# المعمارية الهندسية للنظام (Enterprise System Architecture Guide)

يقدم هذا الدليل نظرة معمارية شاملة على التصميم الداخلي وبنية البرمجيات لنظام **ApexSaaS ERP**، مع التركيز على المعالجات والتحسينات المضافة حديثاً لدعم متطلبات الإفصاح المالي الدولي وحماية عزل البيانات.

---

### 1. الأنماط الهندسية والمعمارية (Architectural Patterns)

تبنى النظام معمارية **المونوليث المعياري القابل للتوسيع (Modular Monolith)** التي تدمج قوة التطوير الموحد مع عزل فيزيائي ومسؤوليات مستقلة لكل موديول مالي أو تشغيلي.

```
       ┌────────────────────────────────────────────────────────┐
       │                 React Single-Page UI                   │
       │     (Customizable BI Dashboard & Reports Engine)        │
       └───────────────────────────┬────────────────────────────┘
                                   │
                     HTTPS REST API / JSON Payload
                                   │
       ┌───────────────────────────▼────────────────────────────┐
       │                Express Core Controller                 │
       │         (RBAC Authorization, Route Gateways)           │
       └───────────────────────────┬────────────────────────────┘
                                   │
               ┌───────────────────┼───────────────────┐
               │                   │                   │
       ┌───────▼───────┐   ┌───────▼───────┐   ┌───────▼───────┐
       │  Accounting   │   │  Human Res.   │   │   Inventory   │
       │  (IAS 7 / 1)  │   │  (Payroll)    │   │  (Logistics)  │
       └───────┬───────┘   └───────┬───────┘   └───────┬───────┘
               │                   │                   │
               └───────────────────┼───────────────────┘
                                   │
                     Drizzle ORM / SQL Dialect
                                   │
       ┌───────────────────────────▼────────────────────────────┐
       │               Enterprise DB Engine (PG)                │
       │      (PostgreSQL / PGlite Client / Migration)          │
       └────────────────────────────────────────────────────────┘
```

#### أ. مبدأ عزل المستأجرين (Tenant Isolation Engine)
* يتم تطبيق عزل المستأجرين على مستوى خادم قواعد البيانات عبر تصفية الاستعلامات بـ `tenant_id`.
* يتم إعداد وتمرير `app.current_tenant_id` في الجلسة المحاسبية النشطة لضمان استحالة حدوث أي تسريب للبيانات الحساسة لشركة شريكة أخرى.

#### ب. فصل الواجهات وعقود البيانات (Separation of Concerns)
* يتم تخزين تصنيفات النماذج المالية والأنواع المشتركة في ملف مركزي مخصص `/src/types.ts` لمنع التكرار وضمان تطابق الأنواع البرمجية (Type-Safety) بنسبة 100% بين الواجهة الأمامية والخلفية.

### 2. محرك التقارير المالية المتوافق مع معايير IFRS

تم تصميم هذا المحرك لخدمة المعايير الدولية واللوائح المحاسبية المصرية الصارمة:

#### أ. معيار المحاسبة الدولي رقم 7 (IAS 7 - التدفقات النقدية)
1. **الطريقة المباشرة (Direct Method):** يحلل محرك الأستاذ العام قيود اليومية التي تحتوي على حركة نقدية في أحد الحسابات (نقدية بالصندوق أو البنوك) ويعمل على مقارنتها وتتبع الحركة المقابلة لها في القيد لتصنيفها كنشاط تشغيلي أو استثماري أو تمويلي، مع إفراز الفئات بشكل موثوق وآلي.
2. **الطريقة غير المباشرة (Indirect Method):** يستخلص صافي الربح التشغيلي كحجر أساس للتقرير، ثم يجري معالجة التسويات الدفترية للأرباح غير المحققة والفروق الجارية للمدينين، والمخزون، والذمم التجارية الدائنة للوصول إلى النقدية الفعلية الناجمة من الأنشطة التشغيلية.

#### ب. معيار المحاسبة الدولي رقم 1 (IAS 1 - عرض القوائم المالية)
* يتتبع الحركات المقيدة في بنود رأس المال والاحتياطيات والمسحوبات الشريكة، ويرحل تلقائياً صافي أرباح الدورة المالية الحالية المنبثقة من حسابات الإيرادات والمصروفات، مما يعطي كشفاً فورياً شاملاً للتغير في حقوق الملكية للمساهمين.

### 3. تخصيص واجهة المستخدم وسلاسة تجربة الاستخدام

تم إعداد واجهة ذكاء الأعمال (**BI Dashboard**) ومحرك التقارير لتقديم مستويات تخصيص فائقة تناسب متطلبات متخذي القرار والمدراء الماليين:

1. **إعادة الترتيب بالسحب والإفلات (Drag & Drop):** تم دمج نظام ترتيب مرن يعتمد على واجهات HTML5 الأصلية لحفظ رغبات المستخدم وترتيب الموديولات بالشكل المناسب له مع تخزين دائم وتلقائي في وحدة التخزين المحلية `localStorage` للمتصفح.
2. **عناصر التحكم الذكية بالإظهار والإخفاء:** مكن النظام مدراء النظام والمدراء الماليين (CFOs) من تعديل رؤيتهم للمؤشرات عبر تصفية كروت الأداء والنسب وتخزين التفضيلات بشكل فوري دون التداخل مع خيارات المستخدمين الآخرين.
3. **تكامل التصدير العالي الدقة (IFRS Compliant Excel Exports):** صممت خوارزمية التصدير لتوليد ملفات متوافقة مع معالجات جداول البيانات لطباعة وتدقيق البيانات بشكل مباشر مع الحفاظ على التنسيقات المالية ولغة التقارير الرسمية.

### 4. بنية وحدة التخزين السحابية للمؤسسات (Enterprise Storage Foundation)

صُممت معمارية التخزين في موديول المستندات والأرشيف الإلكتروني (Document ECM) كبنية تحتية دفاعية ومرنة تضمن أعلى مستويات حماية البيانات وحظر فقدان المستندات تحت أي ظروف تشغيلية:

#### أ. محرك تتابع الفشل التلقائي وعزل الدوائر (Priority-Based Failover & Circuit Breaker)
* يعتمد النظام على سلاسل حيوية لمرونة التخزين مرتبة بالأولية الكبرى:
  `S3 / MinIO ──> Azure Blob Storage ──> Local Filesystem ──> Postgres BYTEA`
* عند رصد أي هبوط أو انقطاع في المزود الرئيسي (مثل انقطاع خدمة S3)، يحول المحرك فوراً وبشكل شفاف دون أي انقطاع للمستخدمين إلى المزود التالي في السلسلة.
* يتضمن النظام **واقي دوائر ذكي (Circuit Breaker)** يقوم بتعطيل أي مزود بشكل مؤقت لمدة 15 ثانية فور تكرار 3 أخطاء متتالية لتخفيف العبء ومنع عرقلة عمليات المعالجة.

#### ب. محرك معالجة الكتل الضخمة (Streaming Chunk Upload Engine)
* صُمم المحرك للتعامل مع ملفات عملاقة تتجاوز 20 جيجابايت باستخدام تقنيات تدفق البيانات المباشرة (Streaming Pipelines) وبمعدل استهلاك ذاكرة ثابت وقريب من الصفر (Zero-RAM Stitching).
* يدعم المحرك التحقق الذاتي لكل كتلة باستخدام خوارزميات التجزئة SHA256، واستعادة واستئناف الرفع (Resume/Pause)، والتنظيف التلقائي للكتل المهجورة (Orphan Chunks).

#### ج. التشفير والضغط الشفاف (Transparent Encryption & Compression)
* جميع البيانات المرفوعة تخضع للضغط التلقائي باستخدام GZIP (للمستندات النصية القابلة للضغط) لتوفير مساحة التخزين.
* يتم تشفير الملفات في لحظة كتابتها (On-the-fly Encryption) باستخدام خوارزميات AES-256-CBC مع توليد نواقل عشوائية (IV) فريدة لكل ملف، ولا تُخزن المفاتيح نهائياً داخل المجموعات المرفوعة لضمان الخصوصية المطلقة.
